Search and Lookup API

Still a work in progress (and thus undocumented), but most of the REST endpoints (eg, /search?q=blood, /id/doi/10.000/whatever, /files/sha1/598eae248c541f1c2c51e4f3785a76dbc62acd6c) can be accessed with an HTTP Accept header of application/json to get JSON. Eg: 

curl -H "Accept: application/json" 'http://localhost:6969/search?q=heart'
